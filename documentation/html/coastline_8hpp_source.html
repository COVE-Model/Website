334  /* @brief Perform sediment transport and evolve coast
335  @details This is the main function call in the model, all else is called from inside
336  this function during a model run. The method begins by calculates the coastline morphology
337  (CalculateMorphology), generates the coastal cells (BuildCellGeometries), identifies
338  shadowed regions of the coast (GetShadows), diffracts waves into the shadowed regions
339  (RefractDiffractShadowZone) and transforms offshore waves to wave breaking (TransformWaves).
340  Subsequently the model calculates alongshore sediment transport following your favourite
341  alongshore sediment transport formula (CalculateFlux) to determine the volume change in
342  each coastal cell, and then calculates the position change of the coast using SolveCubic or
343  SolveQuadratic. Finally IntersectionAnalysis checks for intersections in the coast (e.g.
344  breaching/reattaching) and then the model checks that the spacing of coastal nodes hasn't
345  gotten too big or small (CheckNodeSpacing).
346  @param TimeDelta Double, the model timestep (days)
347  @param TheWave Wave Object sampled from a waveclimate object.
348  @param FluxType Int, 1 = CERC equation, others are yet to be implemented, default is 1.
349  @param RefDiffFlag Int, 0 = Off, 1 = On, default is 1.
350  @param FluxFraction Double, Proportion of sediment input at boundaries when fixed (untested, default is 0).
351  @param LostFluxFraction Double, Proportion of sediment lost offshore during flux (default is 0).
352  @author Martin D. Hurst
353  @date 29/10/2015
354  */
355  void TransportSediment(double &TimeDelta, Wave TheWave, int FluxType=1, int RefDiffFlag=1, double FluxFraction=0, double LostFluxFraction=0);
